## Notes

1. All custom classes are prefixed with `_` (underscore)
2. Classes follow a BEM-like naming convention with underscores
3. Utility classes use abbreviations (e.g., `._mar_t` for margin-top, `._padd_b` for padding-bottom)
4. Component classes use descriptive names (e.g., `._feed_inner_timeline_post_box`)
5. Some classes may have responsive variants (e.g., `._feed_inner_ppl_card` and `._feed_inner_ppl_card_mobile`)
